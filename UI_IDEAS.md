# UI 改善アイデア

目的: 初見のユーザーが「1 Cut / Split → 2 Cut Stock → 3 Edit」というワークフローを、説明なしで視覚的に理解できるようにする。実装前のアイデア集であり、確定仕様ではない。

## 現状の課題

- 3つの作業エリアが同じ配色・同じ見た目で、「素材を作る場所」と「完成品を組む場所」の区別が視覚的にない。
- プレビューが `2 Cut Stock` ペインの中に同居しており、独立した役割(モニタ)に見えない。
- プレビューで「いま何が再生されているか」が変わる(ソース再生 / カット範囲ループ / 編集シーケンス再生)のに、その状態表示が小さなテキスト(`playInfo`)だけ。
- 番号 1 / 2 / 3 はあるが、フローの向き(1で切る → 2に溜まる → 3に並べる)が示されていない。
- ボタンがテキストのみで、機能の当たりがつけにくい。

## 案A: エリアごとのアクセントカラー

各エリアに固有色を割り当て、ヘッダ・選択枠・関連ハイライトを同系色で貫く。「色=工程」を覚えれば迷子にならない。

| エリア | 役割 | 色案 |
|---|---|---|
| 1 Cut / Split | 素材を切る | 青 (#4c8bf5 — 既存の cut band 色を昇格) |
| 2 Cut Stock | 素材が溜まる | ティール / 緑 (#2fbfa3 など) |
| 3 Edit | 完成シーケンス | アンバー (#ffb648 — 既存の選択色を昇格) |

- `pane-head` の左端に色帯(4px 程度)+ ステップ番号を丸バッジ化して同色で塗る。
- 選択・編集モードの色もエリア色に揃える:
  - タイムラインのカット編集(handle / band)= 青系
  - Cut Stock の選択カード = ティール枠
  - Edit のクリップ選択・playhead = アンバー
- 「この material は Edit で使用中」の破線ハイライトはアンバー(Edit 色)にして、"Edit 側と繋がっている" ことを色で示す。
- ドラッグ中のドロップ先ハイライト(insert marker)もアンバーで統一。
- ダークテーマ前提でコントラストを確保する(バッジ文字は黒地に対して AA 目安)。

## 案B: フローの可視化

- ステップバッジ: `① Cut` `② Stock` `③ Edit` のように丸数字+短い動詞ラベル。
- Cut Stock ヘッダに小さな矢印ガイド「✂ Cut → 📦 Stock → 🎞 Edit」を置く、またはエリア境界に薄い → を描く。
- 空状態の placeholder を「次にやること」の指示文に統一(既に一部あり):
  - Cut / Split が空: 「動画を追加して、タイムラインをダブルクリックで切り出し」
  - Cut Stock が空: 「1 のタイムラインで切り出すとここに溜まる」
  - Edit が空: 「2 のカードをここへドラッグ」
- 最初の1回だけの操作(Add Video)は現状の赤点滅を維持しつつ、完了したら次のステップのヒントを点灯させる「順送りガイド」にする。

## 案C: Preview(モニタ)の独立

現在 Horizontal / Vertical プレビューは `2 Cut Stock` ペイン内にあるが、プレビューは「ストック置き場」ではなく全工程共通のモニタ。再生対象が変わることを踏まえ、独立させる。

- `2 Cut Stock`(カード棚)と `Preview / Monitor` を別ペインに分割する。
  - レイアウト案1: 中段を「左: Cut Stock(棚のみ) / 中央: Monitor 16:9 / 右: Vertical 9:16」の3分割(現状に近く、ヘッダだけ分離)。
  - レイアウト案2: Monitor を中央上寄せの大きな1枚にし、16:9 / 9:16 をタブまたはトグルで切替(画面が狭い環境向け)。
- 再生コンテキストをバッジで明示(現在の `playInfo` を昇格):
  - `SOURCE` = ソース素の再生(青)
  - `CUT` = カット範囲ループ(ティール)
  - `EDIT` = 編集シーケンス再生(アンバー)
  - モニタの枠色もこのモード色に同期させ、「いま何を見ているか」を枠の色だけで判断できるようにする。
- Edit 再生中は Edit エリアの playhead・選択クリップと同じアンバーで繋がって見えるようにする。

## 案D: アイコン導入

静的構成を守るため、外部アイコンフォントは使わずインライン SVG か絵文字で。

| ボタン / 場所 | 案 |
|---|---|
| Add Video | ＋🎬(点滅誘導は現状維持) |
| 1 Cut / Split ヘッダ | ✂ |
| 2 Cut Stock ヘッダ | 📦 または 🗂 |
| 3 Edit ヘッダ | 🎞 |
| ExportVideo | ⬇ または 📤 |
| Play / Pause / Loop | ▶ / ⏸ / 🔁(テキスト併記) |
| Undo / Redo | ↩ / ↪ |
| New / Sessions | 🗂 + 「Sessions」 |

- アイコンのみのボタンは作らない(初見理解を優先し、必ず短ラベル併記)。
- 絵文字は環境差(Windows / Mac の見た目差)があるため、最終的にはインライン SVG に寄せるのが安全。まず絵文字で試して評価するのは低コスト。

## 実装するときの推奨順序(段階的)

1. **CSS 変数でエリア色を定義**(`--area-cut` / `--area-stock` / `--area-edit`)+ pane-head の色帯とステップバッジ。DOM 変更ほぼなし、CSS 中心で低リスク。
2. **選択・ハイライト色をエリア色に統一**(cut band / カード選択 / edit クリップ / insert marker)。
3. **playInfo のバッジ化とモニタ枠色の同期**(再生モードの可視化)。
4. **アイコン導入**(まず絵文字、評価後に SVG 化)。
5. **Preview の独立ペイン化**(レイアウト・リサイズ・スプリッタ変更を伴い一番リスクが大きいので最後。案1 → 案2 の順で検討)。

各ステップは独立して出荷可能で、1〜4 までは既存レイアウトを崩さない。
